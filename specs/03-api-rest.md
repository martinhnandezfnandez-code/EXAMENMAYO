# Spec 3: REST API y Presentación

## Requisitos
- Endpoints REST completos
- DTOs con records
- Mappers con MapStruct
- Manejo global de errores
- Validaciones Jakarta

## Acceptance Criteria
- [ ] Endpoints /players: CRUD completo con DTOs
- [ ] Endpoints /characters: CRUD completo con DTOs
- [ ] Endpoints /clans: CRUD completo con DTOs
- [ ] Endpoints /combats: iniciar combate, consultar por ID
- [ ] Endpoints /missions: asignar, completar, listar
- [ ] Endpoints /shop: comprar, vender objetos
- [ ] GlobalExceptionHandler con @RestControllerAdvice
- [ ] DTOs como records con validaciones
- [ ] ResponseEntity en todos los endpoints
- [ ] Tests de integración pasan
- [ ] Código compila sin errores
